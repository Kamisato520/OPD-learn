OPD：让学生自己走路，
老师只在真实踩过的
地方纠偏。

SFT 像背标准答案，RL 像最后看分数。OPD 介于两者之间：学生先按自己的策略生成，教师再沿着学生真实的 rollout 给稠密指导。

先把三件事分清

SFT / OPD / RL：
老师出现的时刻不同。

零基础最容易混的点是：OPD 不是普通蒸馏换个说法。它真正换掉的是训练时的状态分布。

SFT：背答案

  • 数据：固定标准答案。
  • 比喻：抄范文。
  • 弱点：不知道学生自己会在哪里走偏。

OPD：陪练纠偏

  • 数据：学生自己的 rollout。
  • 比喻：老师跟着学生走，边走边指出下一步。
  • 强点：训练时看到的状态更像推理时。

RL：看最后得分

  • 数据：学生尝试结果。
  • 比喻：做完整张卷子才知道对错。
  • 弱点：信号稀疏，credit assignment 难。
核心循环

OPD 的四步，其实就是“先暴露问题，再精准纠偏”。

只要记住这个循环，再看 MiniLLM、GKD、pi-Flow、DiffusionOPD 都会顺很多。

1

学生采样

让学生按当前策略生成，不提前塞教师答案。

2

暴露路径

LLM 是 token 序列；图像模型是去噪或 flow 轨迹。

3

教师评价

教师在学生真实前缀或中间状态上给分布、速度或 reward。

4

学生更新

用 KL、advantage 或 L2 matching，把偏差变成梯度。

最小公式集

只先记四个公式。别急着背全家桶。

公式不是为了炫技，而是给你一个“看到新论文时知道它在改哪里”的坐标系。

1. Reverse KL：别走到老师不认可的地方

$$\mathcal L_{\mathrm{OPD}}=\mathbb E_{y\sim\pi_\theta}\sum_t D_{\mathrm{KL}}\left(\pi_\theta(\cdot|c_t)\|\pi_T(\cdot|c_t)\right)$$

比喻：学生脚下这条路，老师说哪里危险，学生就少往哪里踩。适合压制错误模式。

2. Token advantage：老师比学生更支持，就加强

$$A_t=\log\pi_T(y_t|c_t)-\log\pi_\theta(y_t|c_t)$$

比喻：同一个动作，老师点头越强，学生下次越敢做。缺点是 sampled-token 方差大。

3. GKD：固定答案和学生路径可以混合

$$\mathcal L=(1-\lambda)\mathbb E_{\mathcal D}D+\lambda\mathbb E_{y\sim\pi_\theta}D$$

比喻：先看范文，再上场陪练；\(\lambda\) 越大，越像纯 OPD。

4. Diffusion / Flow：路径从 token 变成轨迹

$$\mathcal L_{\mathrm{traj}}=\mathbb E_{z_t\sim \mathrm{student}}\sum_t \|v_\theta(z_t,t)-v_T(z_t,t)\|_2^2$$

比喻：不是只看最终画作，而是在学生每一步上色/去噪时，老师纠正笔触方向。

图像生成重点

图像生成里的 OPD：老师不只看最终图，而是纠正生成轨迹。

LLM 的路径是 token。扩散和 flow 的路径是 latent 从噪声走向图像的过程。OPD 的核心迁移，就是把“学生 token rollout”换成“学生自己的去噪/ODE/SDE 轨迹”。

DiMO

masked diffusion
解决什么
把 masked diffusion 图像生成压成更少步甚至一步，但避免普通蒸馏分布错位。
OPD 信号
学生在自己的 masked rollout 上做分布匹配，配合 generalized Jeffrey divergence。
怎么理解
像让学生自己补画缺口，再让老师在这些真实补画位置上改。

pi-Flow

flow / few-step
解决什么
few-step flow 既要快，又要保持质量和多样性。
OPD 信号
学生 policy 先走自己的 ODE 子步，再用教师 velocity field 做 L2 imitation。
怎么理解
像学生自己开车过弯，教练在学生实际路线的每个点纠正方向盘角度。

D-OPSD

self-distill T2I
解决什么
微调 few-step 文生图模型时，保留原本的少步生成能力。
OPD 信号
同一模型分成 student 与 teacher：学生只看文本，教师看文本+目标图像，在学生轨迹上匹配预测分布。
怎么理解
像学生闭卷画，老师开卷看参考图，但只在学生已经画出的过程上指点。

DiffusionOPD

multi-task diffusion
解决什么
多任务图像 RL 会互相干扰，串行训练又容易遗忘。
OPD 信号
先独立训练任务专家，再把专家能力沿统一学生的 SDE/ODE rollout 蒸馏进一个模型。
怎么理解
像先请多个专科老师各教一门，再让总教练在学生自己的训练路线中整合。
最新进展地图

OPD 的演化方向：从语言 token，到图像轨迹。

这不是论文百科，只保留你需要建立方向感的节点。

2023

MiniLLM / GKD

确立学生 rollout + 教师 token guidance 的基础范式。

2025

DiMO / pi-Flow

OPD 开始进入图像生成：masked diffusion 与 flow 轨迹蒸馏。

2026 Q1

工业 MOPD

MiMo、DeepSeek、GLM 把 OPD 用作多教师/多阶段能力整合。

2026 Q2

D-OPSD

文生图少步模型用自蒸馏微调，学生走自己的生成轨迹。

2026 Q2

DiffusionOPD

把 OPD 统一到 diffusion Markov process，覆盖 SDE 与 ODE。

下一步

读图像线

优先 pi-Flow → D-OPSD → DiffusionOPD，再回看基础 OPD。

BASE

先读 GKD

最容易把 OPD 写成训练循环，不容易被 policy-gradient 细节卡住。

记住：学生样本比例 \(\lambda\) 是从 KD 走向 OPD 的旋钮。

BASE

再读 Rethinking

它告诉你不是所有强老师都可教，关键看 teacher-student overlap。

记住：高分教师不等于好教师。

IMAGE

优先读 pi-Flow

它把“学生自己的轨迹”讲得最像图像生成里的 OPD。

记住：teacher velocity 要在 student trajectory 上匹配。

IMAGE

再读 DiffusionOPD

它把 OPD 从离散 token 推到连续 diffusion process。

记住：图像 OPD 的 KL 经常会变成 mean / velocity matching。

SYSTEM

实现看 verl / KDFlow

真正的瓶颈通常是 rollout、teacher forward、logits/hidden state 传输。

记住：算法能跑和规模能跑是两件事。

IMAGE

最后看 D-OPSD

它把 self-distillation 搬到 few-step T2I 微调上。

记住：teacher 的 privilege 是 text + target image。

你现在只需要带走一句话。

OPD 的本质不是“老师更强”，而是“老师在学生真实会走到的地方教”。图像生成里，这个地方就是学生自己的去噪或 flow 轨迹。
