先分清训练范式

SFT、KD、OPD、RL 的差别，是训练时看到的状态不同。

OPD 的关键不只是“有教师”。关键是教师在学生自己产生的状态上给指导，所以它比 SFT/KD 更贴近推理时分布，又比纯 RL 多了稠密信号。

SFT

背标准答案

数据来自人类或教师固定答案。像学生抄范文，学得快，但不知道自己独立写作时会在哪里跑偏。

公式：\(-\log\pi_\theta(y^\*|x)\)

Offline KD

看老师路线

在教师或数据集轨迹上拟合教师分布。比 SFT 更软，但训练状态仍主要来自老师，不是学生自己。

风险：exposure bias 仍在。

OPD

陪练纠偏

学生先走自己的 rollout，教师沿这些状态给 token 或轨迹级指导。像教练坐副驾驶，按学生实际路线纠正。

核心：\(y\sim\pi_\theta\)。

RL

最后看分数

学生尝试完整输出，再由 reward 或 verifier 打分。信号强但稀疏，credit assignment 更难。

OPD 可视为 dense reward RL。

我的统一理解

OPD 不是一个 loss，而是一种训练状态分布的切换。

把它想成开车教学：SFT 让你背路线，KD 让你看老师怎么开，RL 等你开完再评分；OPD 是教练坐在副驾驶，在你真实走到的路口纠正方向。

core thesis

学生先暴露问题，老师再给局部纠偏。

所以 OPD 的价值不在“教师更强”本身，而在教师能否在学生真实访问到的状态上给出可靠、稠密、可优化的信号。

我会用四个问题判断一篇 OPD 论文

谁采样？如果轨迹不是学生自己采样，就更像离线 KD；如果是学生 rollout，才真正触及 exposure bias。
老师能给什么？白盒给 logits / top-k；黑盒给偏好 reward；自蒸馏给 privileged context；图像模型给 velocity、score 或 mean。
信号落在哪？token、prefix、trajectory step、latent state、整段答案，决定了 credit assignment 难不难。
怎么防崩？高熵门控、top-k support、KL 约束、teacher routing、anchor regularization，本质都是在管教师噪声和学生方差。
核心公式地图

看到新 OPD 论文，先问它改了哪一块。

一篇 OPD 论文通常在四处做文章：sampling policy、teacher access、loss scope、stability mechanism。

打开公式推导子页面

1. Student rollout

$$y=(y_1,\dots,y_T)\sim\pi_\theta(\cdot|x),\quad c_t=(x,y_{<t})$$

在改什么：把训练分布从数据/教师轨迹切到学生轨迹。

学生先暴露自己的路径。LLM 是 token prefix；扩散模型是 latent/denoising state。

2. Reverse KL

$$\mathcal L_{\mathrm{RKL}}=\mathbb E_{y\sim\pi_\theta}\sum_t D_{\mathrm{KL}}(\pi_\theta(\cdot|c_t)\|\pi_T(\cdot|c_t))$$

在改什么：压掉学生放到教师低概率区域的错误模式。

mode-seeking：惩罚学生把概率放到教师不认可的区域，常用于压错误模式。

3. Forward KL

$$D_{\mathrm{KL}}(\pi_T\|\pi_\theta)=\sum_v\pi_T(v|c)\log\frac{\pi_T(v|c)}{\pi_\theta(v|c)}$$

在改什么：让学生覆盖教师认为可能的多个答案。

mode-covering：保留教师认为可能的多种选择，Entropy-Aware OPD 在高熵位置使用它。

4. Sampled-token advantage

$$A_t=\log\pi_T(y_t|c_t)-\log\pi_\theta(y_t|c_t)$$

在改什么：把 full-vocab KL 近似成采样 token 的增减权重。

老师比学生更支持这个 token，就增加它；反之降低它。优点省算力，缺点方差高。

5. GKD 混合目标

$$\mathcal L=(1-\lambda)\mathbb E_{\mathcal D}D+\lambda\mathbb E_{y\sim\pi_\theta}D$$

在改什么：用 \(\lambda\) 控制离线稳定性和 on-policy 真实性。

\(\lambda\) 是从离线 KD 走向纯 OPD 的旋钮，工程上比纯 policy-gradient 更稳。

6. ExOPD dense reward

$$r_T(x,y)=\log\frac{\pi_\*(y|x)}{\pi_{\mathrm{ref}}(y|x)},\quad \lambda>1\Rightarrow reward\ extrapolation$$

在改什么：把教师分布解释成 dense reward，而不是只当模仿目标。

把教师看成 reward 函数。普通 OPD 是模仿，ExOPD 尝试沿教师偏好继续外推。

7. Black-box reward

$$\max_\theta\ \mathbb E_{y\sim\pi_\theta}D_\phi(x,y)$$

在改什么：把不可见的教师 logits 替换成可学习的偏好评分。

没有教师 logits 时，用判别器、rubric 或 verbal score 把教师偏好变成 reward。

8. Diffusion / flow trajectory

$$\mathcal L_{\mathrm{traj}}=\mathbb E_{z_t\sim \mathrm{student}}\sum_t\|v_\theta(z_t,t)-v_T(z_t,t)\|_2^2$$

在改什么：把 token 分布匹配搬到连续 latent 轨迹匹配。

图像生成里，token KL 常变成速度场、均值、score 或 flow map 的轨迹匹配。

公式谱系：从背答案到轨迹纠偏

SFT NLL\(-\log\pi_\theta(y^\*|x)\)：只学标准答案。
Offline KD\(D(\pi_T,\pi_\theta)\)：学教师分布，但状态多来自教师/数据。
On-policy KL\(y\sim\pi_\theta\)：学生走到哪，老师纠到哪。
Advantage / Reward\(\log\pi_T-\log\pi_\theta\)：把教师偏好变成局部奖励。
Diffusion trajectory\(z_t\sim\mathrm{student}\)：在学生 latent 轨迹上匹配速度、均值或 score。

1

Survey

先建立分类：白盒、黑盒、自蒸馏、上下文、工业多教师。记住：OPD 是训练状态分布的改变。

2

Foundations

MiniLLM、GKD、ExOPD。解决“目标函数怎么写”和“它为什么像 dense RL”。

3

Practical intuition

Thinking Machines 博客。把公式翻译成工程直觉：学生在哪错，老师就在哪教。

4

Breaks

Revisiting、Entropy-Aware、Rethinking。学失败模式：方差、高熵坍缩、教师不可教。

5

No logits

Black-Box OPD。没有教师 logits 时，用判别器或 reward 近似教师偏好。

6

No teacher

OPSD、SDFT。老师可以是同一个模型的 privileged view，而不是外部大模型。

7

Context

OPCD、OEL。把 context、经验、系统提示蒸进模型权重。

8

Industrial

Qwen3、DeepSeek、MiMo、GLM。看 OPD 如何变成多阶段 post-training glue。

01 · Survey

OPD Survey

PDFarXiv
研究问题
OPD 文献从白盒 logits 扩展到黑盒 reward、自蒸馏、上下文蒸馏和工业多教师，需要统一坐标。
核心方法
按 teacher access、feedback granularity、loss scope、training distribution 分类。
关键公式
\(\mathbb E_{y\sim\pi_\theta}\sum_t\mathcal D(\pi_T(\cdot|c_t),\pi_\theta(\cdot|c_t))\)
公式到底在改什么
把“固定数据上的监督”改成“学生访问状态上的教师距离”。
论文结论 / 实验发现
OPD 不是某个单一算法，而是一族围绕学生 rollout 设计 teacher signal 的训练范式。
我的归纳
读新论文先定位四个轴：谁采样、老师给什么、loss 作用在哪、稳定性怎么做。
局限或风险
Survey 给地图，不给可直接复现的工业 recipe。
一句话记忆法
这是 OPD 的地图，不是交通工具。
02 · Foundation

MiniLLM

PDFarXiv
研究问题
固定轨迹 KD 会让学生只学老师走过的路，推理时一旦自己走偏，训练中没有见过这种前缀。
核心方法
学生 on-policy 生成，教师在学生前缀上给 reverse-KL / log-prob signal，用 policy-gradient 式目标训练学生。
关键公式
\(\min_\theta D_{\mathrm{KL}}(q_\theta(y|x)\|p_T(y|x))\)，常用 \(A_t=\log p_T(y_t|c_t)-\log p_\theta(y_t|c_t)\)。
公式到底在改什么
把 distillation 的分布从 teacher/data trajectory 换成 student trajectory，并用 reverse KL 压低学生错误模式。
论文结论 / 实验发现
生成式 LLM 蒸馏中，on-policy + reverse KL 比传统离线 KD 更能处理学生自己的生成偏差。
我的归纳
MiniLLM 是 OPD 的入门原型：它说明“让学生自己犯错”不是噪声，而是训练数据的核心。
局限或风险
sampled-token estimator 方差高，长序列 credit assignment 不稳。
一句话记忆法
第一个把“陪练纠偏”讲成 LLM 蒸馏算法的人。
03 · Foundation

GKD

PDFarXiv
研究问题
纯 on-policy 有成本和稳定性问题，纯 off-policy 又有 exposure bias，如何统一二者。
核心方法
用 \(\lambda\) 混合数据轨迹和学生轨迹，token-level divergence 可选 KL、reverse KL、JSD/GJSD。
关键公式
\(\mathcal L=(1-\lambda)\mathbb E_{(x,y)\sim\mathcal D}D+\lambda\mathbb E_{y\sim\pi_\theta}D\)
公式到底在改什么
\(\lambda\) 控制训练样本到底多大比例来自学生自己，是从 KD 过渡到 OPD 的旋钮。
论文结论 / 实验发现
在多类生成任务上，混合学生生成与教师监督能比单纯模仿数据轨迹更稳地提升学生。
我的归纳
工程上先从 GKD 入门，因为它不要求你一次跳到纯 RL 或复杂 PG，能渐进增加 on-policy 比例。
局限或风险
仍需要教师 logits，full-vocab divergence 对大词表模型昂贵。
一句话记忆法
\(\lambda\) 是 KD 到 OPD 的音量旋钮。
04 · Dense RL

ExOPD / G-OPD

PDFarXiv
研究问题
OPD 是否只能把学生推向教师，还是能把教师信号解释成 reward 后继续外推。
核心方法
把 teacher-reference log-ratio 看成 dense reward，并用 \(\lambda\) 控制 imitation、distillation 和 extrapolation。
关键公式
\(r_T(x,y)=\log\frac{\pi_\*(y|x)}{\pi_{\mathrm{ref}}(y|x)}\)，\(\max_\theta \mathbb E_{\pi_\theta}[\lambda r_T-\beta\log\frac{\pi_\theta}{\pi_{\mathrm{ref}}}]\)
公式到底在改什么
把 teacher distribution 从“要拟合的目标”改成“可优化的 dense reward”。
论文结论 / 实验发现
OPD 可以放进 KL-constrained RL 视角；\(\lambda>1\) 的 ExOPD 尝试利用教师偏好超越教师。
我的归纳
这篇最适合理解 OPD 与 RL 的关系：OPD 是 dense reward RL 的一个低方差特例。
局限或风险
外推会放大教师偏差和校准错误，不能把 \(\lambda>1\) 当免费午餐。
一句话记忆法
模仿老师，也可以顺着老师偏好再走一步。
05 · Stability

Revisiting OPD

PDFarXiv
研究问题
为什么 sampled-token OPD 在长序列、多任务和特殊 token 上容易不稳。
核心方法
从单个采样 token 的 advantage，转向教师 top-K support matching，并处理特殊 token 与局部归一化。
关键公式
\(D_{\mathrm{KL}}(\tilde\pi_\theta^S(\cdot|c_t)\|\tilde q_T^S(\cdot|c_t))\)，其中 \(S\) 是教师 top-K 支持集。
公式到底在改什么
把“一次采样的 token 反馈”改成“教师候选集合的分布反馈”。
论文结论 / 实验发现
OPD 的不稳定很大一部分来自高方差估计和不合适的 token scope，top-K 分布匹配更稳。
我的归纳
这篇告诉你：OPD 不是只要 on-policy 就赢，teacher signal 的局部形状也很关键。
局限或风险
需要 logits/top-K；如果教师在学生偏离前缀上本身不可靠，support matching 也会继承噪声。
一句话记忆法
别只听老师对一个词的态度，要看老师的候选名单。
06 · Stability

Entropy-Aware OPD

PDFarXiv
研究问题
reverse KL 的 mode-seeking 会不会把教师高熵状态下的多样性压成单一路径。
核心方法
按教师 entropy 做门控：低熵位置用 reverse KL，高熵位置补 forward KL 或覆盖项。
关键公式
\(\mathcal L_{\mathrm{EOPD}}=\mathcal L_{\mathrm{RKL}}+\mathbf1[H(\pi_T)>\tau]\mathcal L_{\mathrm{FKL}}\)
公式到底在改什么
让 loss 的方向随教师不确定性变化：确定时收敛，不确定时保多样。
论文结论 / 实验发现
只用 reverse KL 容易在高熵 token 上过度坍缩；entropy-aware 门控能改善多样性和稳定性。
我的归纳
这篇是 OPD 里的“老师也会犹豫”原则：老师犹豫时，学生不该被强迫只学一个答案。
局限或风险
高熵可能代表合理多样性，也可能代表教师混乱；阈值和任务相关。
一句话记忆法
老师犹豫时，别逼学生只选一个答案。
07 · Diagnosis

Rethinking OPD

PDFarXiv
研究问题
为什么更强教师不一定更适合弱学生，OPD 什么时候会失效。
核心方法
监控 teacher/student top-k overlap、entropy gap、teacher novelty，诊断“能不能教”和“学生听不听得懂”。
关键公式
\(M_{\mathrm{overlap}}=\mathbb E_t\frac{|S_T(c_t)\cap S_S(c_t)|}{k}\)
公式到底在改什么
它不直接改 loss，而是给 OPD 增加可观测的匹配度指标。
论文结论 / 实验发现
OPD 的收益依赖教师新颖性和学生可吸收性的平衡，强教师与弱学生差距过大反而可能降低学习效率。
我的归纳
选教师不是选榜单第一，而是选“学生当前阶段能理解、又能提供增量”的教师。
局限或风险
诊断需要 logits/top-k；不同任务的合理阈值不通用。
一句话记忆法
好老师不是最强，而是学生听得懂且有新东西。
08 · Black-box

Black-Box OPD / GAD

PDFarXiv
研究问题
教师只有 API 输出，没有 logits，传统 white-box OPD 无法直接计算 token KL。
核心方法
训练判别器区分教师与学生输出，或把教师偏好转成 reward，再对学生做 on-policy 更新。
关键公式
\(\max_\theta\mathbb E_{y\sim\pi_\theta}D_\phi(x,y)-\beta D_{\mathrm{KL}}(\pi_\theta\|\pi_{\mathrm{ref}})\)
公式到底在改什么
把不可见的 token-level teacher distribution 换成可学习的 sequence-level reward。
论文结论 / 实验发现
即使拿不到 logits，也能保留“学生 rollout + 教师偏好纠偏”的 OPD 精神，但更接近 RL 系统。
我的归纳
黑盒 OPD 的核心瓶颈从“蒸馏 loss”转移到“reward/判别器是否可信”。
局限或风险
判别器可能 reward hacking，且信号通常不如 white-box token logits 稠密。
一句话记忆法
拿不到老师的概率表，就训练一个裁判学老师品味。
09 · Self

OPSD

PDFarXiv
研究问题
没有外部强教师时，模型能否从自己的 privileged view 中获得 token-level 教师信号。
核心方法
学生只看问题并 rollout；教师是同一模型但额外看到参考答案、推理轨迹或 privileged context。
关键公式
\(D(p_T(\cdot|x,y^\*,\hat y_{<t})\|p_S(\cdot|x,\hat y_{<t}))\)
公式到底在改什么
教师不再是外部大模型，而是“开卷视角”的同模型。
论文结论 / 实验发现
privileged context 可以把稀疏正确性信号转成学生 rollout 上的 dense 自蒸馏信号。
我的归纳
OPSD 的启发很适合图像方向：text-only 学生可以从 text+image 教师视角学习中间轨迹。
局限或风险
privileged context 质量决定上限，可能压制学生探索或复制错误解释。
一句话记忆法
同一个学生，开卷版教闭卷版。
10 · Self

SDFT

PDFarXiv
研究问题
持续学习中，如何把 in-context demonstration 的能力内化到权重，同时减少遗忘。
核心方法
同模型 demonstration-conditioned teacher 或 EMA teacher 监督 no-demo student 的 rollout。
关键公式
\(\mathbb E_{y\sim\pi_\theta(\cdot|x)}\log\frac{\pi_\theta(y|x)}{\pi_T(y|x,c)}\)
公式到底在改什么
把“每次推理都带示例”改成“训练时用示例教师，推理时学生不带示例”。
论文结论 / 实验发现
自蒸馏能把 demonstration 的行为模式写入参数，并在 continual tuning 中缓解旧能力掉落。
我的归纳
SDFT 是“把上下文能力压进权重”的一条实用路线，和 OPCD/OEL 是同一思想族。
局限或风险
前提是模型已经能利用 demonstration；否则教师视角本身不强。
一句话记忆法
先让模型带小抄做，再让它不带小抄也会做。
11 · Context

OPCD

PDFarXiv
研究问题
能否把系统提示、知识库、长上下文或工具经验蒸进模型权重，减少推理时上下文依赖。
核心方法
学生在无 context 条件下 rollout，教师带 context 在同一学生前缀上提供分布指导。
关键公式
\(\mathbb E_{y\sim\pi_\theta(\cdot|x)}\sum_tD_{\mathrm{KL}}(\pi_\theta(\cdot|x,y_{<t})\|\pi_T(\cdot|x,c,y_{<t}))\)
公式到底在改什么
把 context 从推理输入变成训练教师条件，最终让学生无 context 也接近有 context 行为。
论文结论 / 实验发现
context distillation 可以把外部上下文能力压缩到模型内部，降低部署时 prompt/context 成本。
我的归纳
OPCD 是 OPD 的一个很重要变体：老师强不是因为参数更大，而是因为“看得更多”。
局限或风险
会固化错误或过时 context；隐私、更新和遗忘管理很关键。
一句话记忆法
把提示词从输入栏搬进模型脑子。
12 · Experience

OEL

PDFarXiv
研究问题
部署后没有服务端环境或显式 reward，如何从用户端交互中持续学习。
核心方法
从交互轨迹抽取 experience，逐步累积，再把 experience 作为教师 context 做 OPCD consolidation。
关键公式
\(e_i=[e_{i-1};e'_i]\)，再用 \(D(\pi_S(\cdot|x,y_{<t})\|\pi_T(\cdot|x,e_i,y_{<t}))\) 蒸馏。
公式到底在改什么
把在线经验从“只存在日志里”改成“可反复提供教师视角的 context”。
论文结论 / 实验发现

我的归纳

局限或风险
experience extraction、隐私、污染和错误反馈积累是主要风险。

图像生成 / 扩散 OPD 专区

把 token rollout 换成 denoising / flow / latent trajectory。

图像 OPD 的核心转译：学生不是逐 token 生成，而是在 latent 空间里从噪声走向图像。教师信号也从 token logits 变成速度场、均值、score、flow map 或 reward。

LLM 到图像生成的对应关系

token prefix对应 diffusion / flow 的 \(z_t\)、denoising state、ODE/SDE state 或 latent update history。
teacher logits对应 teacher velocity field、score、Gaussian mean、flow map、reward 或 preference model。
KL on token对应 per-step KL、mean matching、velocity L2、trajectory consistency 或 reward-regularized update。
my view

图像 OPD 的关键不是“老师给好图打分”。

关键是老师在学生自己的中间轨迹上持续纠偏：学生在哪个 latent state 走歪，教师就在那个 state 给速度、均值、score 或 reward 方向。

DiMO · 2025

Masked diffusion one-step

问题
多步 masked diffusion 很慢，一步化又缺中间监督。
信号
token-level distribution matching + auxiliary model + generalized Jeffrey divergence。
公式类比
\(D_{\mathrm{GJ}}(p_T(\cdot|z_t),p_\theta(\cdot|z_t))\)：离散图像 token 上的 OPD。
我的判断
DiMO 说明图像 OPD 不只属于连续 diffusion，离散 masked token 也能做学生轨迹纠偏。
pi-Flow · 2025/2026

沿学生 ODE 轨迹匹配速度

问题
few-step flow 常有质量-多样性 trade-off。
信号
学生 policy 产生未来子步速度，教师 velocity field 在学生轨迹上用 L2 匹配。
公式类比
\(\mathbb E_{z_t\sim\mathrm{student}}\|v_\theta(z_t,t)-v_T(z_t,t)\|_2^2\)。
我的判断
这是最容易理解的图像 OPD：老师不是看终点图，而是沿学生 ODE 路线纠正方向盘。
D-OPSD · 2026

text-only 学生 vs text+image 教师

问题
few-step T2I 直接 SFT 会破坏少步推理能力。
信号
同模型双视角：学生只看文本，教师看文本+目标图像，在学生 rollout 上匹配预测分布或速度。
公式类比
\(D(\pi_{\mathrm{text}}(\cdot|z_t),\pi_{\mathrm{text+image}}(\cdot|z_t))\)。
我的判断
它把 OPSD 的开卷/闭卷思想搬到图像：参考图像就是 privileged context。
Flow-OPD · 2026

多任务 flow alignment

问题
多 reward RL 有稀疏奖励、指标跷跷板和 reward hacking。
信号
任务专家先单独 RL，再通过学生 SDE rollout 的 dense trajectory supervision 合并。
公式类比
\(\sum_i w_iD_{\mathrm{KL}}(p_\theta(z_{t-1}|z_t)\|p_{E_i}(z_{t-1}|z_t))\)。
我的判断
它把“多专家合并”从 LLM 搬到 flow model，难点变成专家冲突和轨迹锚定。
DiffusionOPD · 2026

连续状态 Markov OPD

问题
多任务 diffusion RL 联合训练干扰，串行训练易遗忘。
信号
在学生 rollout 轨迹上做 per-step KL / mean matching，统一 SDE 和 ODE refinement。
公式类比
\(\sum_tD_{\mathrm{KL}}(p_\theta(z_{t-1}|z_t)\|p_T(z_{t-1}|z_t))\)。
我的判断
这是图像 OPD 的理论桥：把离散 token prefix 推广成连续 Markov denoising state。
AnyFlow / LiveTalk

视频与实时交互

问题
少步视频生成常丢失 test-time scaling 或实时交互能力。
信号
AnyFlow 用 flow-map transition 学习；LiveTalk 改进 multimodal 条件下的 on-policy distillation recipe。
公式类比
\(\|F_\theta(z_t,t,\Delta t)-F_T(z_t,t,\Delta t)\|\)：匹配学生当前状态的流映射。
我的判断
视频 OPD 更重视完整时空轨迹和条件输入质量，不能只看单帧 reward。
