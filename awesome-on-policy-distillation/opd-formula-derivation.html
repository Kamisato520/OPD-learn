0. 最小数学地基

模型的本质：给“下一步”分配概率。

如果你只记一件事：语言模型不是一次生成整篇答案，而是一步一步选择 token。图像 diffusion / flow 也类似，只是“下一步”发生在 latent 空间。

概率

一个上下文，一个分布

给定前缀 \(c_t=(x,y_{<t})\)，学生模型输出 \(\pi_\theta(\cdot|c_t)\)，教师输出 \(\pi_T(\cdot|c_t)\)。

$$\sum_v \pi_\theta(v|c_t)=1,\quad \pi_\theta(v|c_t)\ge 0$$

直觉：同一道题走到同一个路口，学生和老师各自给一张“下一步概率表”。

序列概率

整段答案是逐步相乘

一个答案 \(y=(y_1,\dots,y_T)\) 的概率来自每一步 token 概率的乘积。

$$\pi_\theta(y|x)=\prod_{t=1}^T \pi_\theta(y_t|x,y_{<t})$$

所以长答案一旦前面走偏，后面看到的状态就和训练数据不同。

负对数似然

乘法变加法

训练时常最小化负 log 概率，因为 log 把乘积变成每一步损失相加。

$$-\log\pi_\theta(y|x)=-\sum_{t=1}^T\log\pi_\theta(y_t|x,y_{<t})$$

这就是 SFT 的起点：让标准答案每一步概率变大。

1. 从 SFT 推到 exposure bias

SFT 学的是标准路线，不是学生真实会走到的路线。

SFT 的公式很自然，但它默认训练时的前缀来自标准答案。推理时前缀来自学生自己，这就是后面 OPD 要修的地方。

SFT objective

从最大似然到 NLL

数据集中有标准答案 \(y^\*\)。最大化它的概率，等价于最小化负 log 概率。

$$\max_\theta\ \log\pi_\theta(y^\*|x) \quad\Longleftrightarrow\quad \min_\theta\ -\sum_t\log\pi_\theta(y_t^\*|x,y^\*_{<t})$$

结论：SFT 的监督状态是 \(c_t^\*=(x,y^\*_{<t})\)。

问题出在哪里？

推理时，学生看到的是 \(c_t=(x,y_{<t})\)，其中 \(y_{<t}\sim\pi_\theta\)。如果学生第 3 步走错，第 4 步的状态就不是训练里的标准前缀。SFT 没告诉它“你已经走歪时该怎么办”。

2. 从 SFT 到 KD

KD 的第一步：别只学一个答案，学老师的概率表。

教师不是只说“正确 token 是 A”，而是给每个 token 一个概率。这样学生能学到相似词、犹豫程度和多解结构。

cross entropy

教师分布监督学生

在同一上下文 \(c\) 上，教师分布是 \(\pi_T\)，学生分布是 \(\pi_\theta\)。让学生给教师高概率 token 也高概率：

$$H(\pi_T,\pi_\theta)=-\sum_v\pi_T(v|c)\log\pi_\theta(v|c)$$

交叉熵可拆成教师自己的熵加 forward KL：

$$H(\pi_T,\pi_\theta)=H(\pi_T)+D_{\mathrm{KL}}(\pi_T\|\pi_\theta)$$

因为 \(H(\pi_T)\) 与学生无关，最小化交叉熵就是最小化 \(D_{\mathrm{KL}}(\pi_T\|\pi_\theta)\)。

offline KD

传统 KD 仍然可能离线

如果上下文来自数据或教师答案，KD 目标是：

$$\mathcal L_{\mathrm{KD}}=\mathbb E_{(x,y)\sim\mathcal D}\sum_tD_{\mathrm{KL}}(\pi_T(\cdot|x,y_{<t})\|\pi_\theta(\cdot|x,y_{<t}))$$

结论：KD 比 SFT 软，但状态仍不一定是学生自己会访问的状态。

3. 推出 OPD 核心公式

只改一个东西：期望从数据分布换成学生 rollout。

这是 OPD 最核心的数学动作。老师仍然给分布指导，但老师评估的前缀来自学生自己。

student rollout

先让学生自己走

学生按当前策略生成整段答案：

$$y=(y_1,\dots,y_T)\sim\pi_\theta(\cdot|x),\quad c_t=(x,y_{<t})$$

然后教师在这些真实前缀上给分布。

$$\pi_T(\cdot|c_t),\quad \pi_\theta(\cdot|c_t)$$

这一步让训练状态和推理状态对齐。

OPD objective

在学生状态上做分布匹配

于是得到通用 OPD 目标：

$$\mathcal L_{\mathrm{OPD}}(\theta)= \mathbb E_{x\sim\mathcal D,\ y\sim\pi_\theta(\cdot|x)} \sum_{t=1}^{T}\mathcal D\!\left(\pi_T(\cdot|c_t),\pi_\theta(\cdot|c_t)\right)$$

结论：OPD = student rollout distribution + teacher dense guidance。

一个极小例子：为什么 on-policy 有价值

假设标准答案前缀是“解方程”，但学生生成了“猜答案”。离线 KD 只在“解方程”后教学生下一步；OPD 会让教师在“猜答案”这个错误前缀上告诉学生：下一步应该回到推理或校验。

这不是数学 trick，而是数据分布变了：训练开始覆盖学生真实错误。

4. KL 方向：为什么 reverse KL 很重要

forward KL 和 reverse KL 看起来只换了顺序，行为完全不同。

这正是 MiniLLM、GKD、Entropy-Aware OPD 等论文反复讨论的核心。

forward KL

覆盖老师可能的答案

$$D_{\mathrm{KL}}(\pi_T\|\pi_\theta)= \sum_v\pi_T(v|c)\log\frac{\pi_T(v|c)}{\pi_\theta(v|c)}$$

如果教师认为某 token 有概率，而学生给接近 0，会被严重惩罚。

倾向：mode-covering，保多样性。

reverse KL

压掉学生不该走的路

$$D_{\mathrm{KL}}(\pi_\theta\|\pi_T)= \sum_v\pi_\theta(v|c)\log\frac{\pi_\theta(v|c)}{\pi_T(v|c)}$$

如果学生把概率放到教师很不认可的 token 上，会被严重惩罚。

倾向：mode-seeking，压错误模式。

两 token 数字直觉

教师认为 \(A=0.9,B=0.1\)。如果学生给 \(A=0.5,B=0.5\)，reverse KL 会明显惩罚学生把过多概率放到 \(B\)；forward KL 则更强调别漏掉教师认为可能的 \(A\) 和 \(B\)。

KL 方向最怕什么适合什么
\(D_{\mathrm{KL}}(\pi_T\|\pi_\theta)\)学生漏掉教师可能答案多解、多样性、保覆盖
\(D_{\mathrm{KL}}(\pi_\theta\|\pi_T)\)学生走到教师低概率区域压错误、强到弱压缩、稳定输出
5. 从 reverse KL 推到 sampled-token advantage

老师比学生更支持这个 token，就增加它；否则降低它。

这是 MiniLLM 和多种工业 recipe 里的重要直觉。严格 full-vocab KL 很贵，所以常把信号压到学生实际采到的 token 上。

start from reverse KL

把 token 看成从学生采样

单个状态 \(c_t\) 的 reverse KL 是：

$$D_{\mathrm{KL}}(\pi_\theta\|\pi_T)= \mathbb E_{y_t\sim\pi_\theta(\cdot|c_t)} \left[\log\pi_\theta(y_t|c_t)-\log\pi_T(y_t|c_t)\right]$$

把括号取负，就得到“教师相对学生更支持多少”的量：

$$A_t=\log\pi_T(y_t|c_t)-\log\pi_\theta(y_t|c_t)$$
policy-gradient view

用 advantage 更新采样 token

最小化 reverse KL 可以近似写成增强教师更认可的采样 token：

$$\nabla_\theta \mathcal L_{\mathrm{RKL}} \approx -\mathbb E_{y_t\sim\pi_\theta} \left[A_t\nabla_\theta\log\pi_\theta(y_t|c_t)\right]$$

结论：\(A_t>0\) 增加该 token；\(A_t<0\) 降低该 token。

caution

为什么后续论文要修它

sampled-token advantage 便宜，但只看一个采样 token，方差高。Revisiting OPD 改为 top-K support matching；Entropy-Aware OPD 在教师高熵位置补 forward KL；Rethinking OPD 用 overlap 和 entropy gap 判断教师是否可教。

6. 推出 GKD 混合目标

GKD 是一个旋钮：从离线 KD 慢慢拧到纯 OPD。

如果一开始纯学生 rollout 不稳，可以混合数据轨迹和学生轨迹。这个思想非常适合工程入门。

two data sources

离线状态 + 学生状态

离线 KD 的期望来自数据：

$$\mathbb E_{(x,y)\sim\mathcal D}\sum_tD(\pi_T,\pi_\theta)$$

OPD 的期望来自学生：

$$\mathbb E_{x\sim\mathcal D,\ y\sim\pi_\theta}\sum_tD(\pi_T,\pi_\theta)$$
mixture objective

用 \(\lambda\) 混合

$$\mathcal L_{\mathrm{GKD}}= (1-\lambda)\mathbb E_{(x,y)\sim\mathcal D}\sum_tD_t +\lambda\mathbb E_{x\sim\mathcal D,\ y\sim\pi_\theta}\sum_tD_t$$

\(\lambda=0\) 是离线 KD；\(\lambda=1\) 是纯 OPD。

7. 从 KL-constrained RL 推到 ExOPD

OPD 也可以从 RL 推出来：教师分布就是一个 dense reward。

这一步很关键：它解释了为什么 OPD 处在 SFT/KD 和 RL 之间。

1

从 KL 约束 RL 开始

我们希望学生拿高 reward，但不要离 reference 太远：

$$\max_{\pi}\ \mathbb E_{y\sim\pi}\left[r(y)\right]-\beta D_{\mathrm{KL}}(\pi(y|x)\|\pi_{\mathrm{ref}}(y|x))$$
2

最优策略的形状

这个优化的解满足：

$$\pi^\*(y|x)\propto \pi_{\mathrm{ref}}(y|x)\exp\left(\frac{r(y)}{\beta}\right)$$

直觉：reference 给底盘，reward 把好答案指数级放大。

3

让最优策略等于教师

如果我们想让 \(\pi^\*=\pi_T\)，只要设置：

$$r_T(y|x)=\beta\log\frac{\pi_T(y|x)}{\pi_{\mathrm{ref}}(y|x)}$$

代回去得到 \(\pi^\*\propto\pi_{\mathrm{ref}}\frac{\pi_T}{\pi_{\mathrm{ref}}}=\pi_T\)。

4

ExOPD：缩放教师 reward

如果把 reward 乘以 \(\lambda\)：

$$\pi_\lambda(y|x)\propto \pi_{\mathrm{ref}}(y|x)\exp\left(\lambda\log\frac{\pi_T(y|x)}{\pi_{\mathrm{ref}}(y|x)}\right) =\pi_T(y|x)^\lambda\pi_{\mathrm{ref}}(y|x)^{1-\lambda}$$

\(\lambda=1\) 回到教师；\(\lambda<1\) 是保守蒸馏；\(\lambda>1\) 是沿教师偏好外推。

8. 黑盒 OPD 的推导位置

拿不到 logits 时，KL 公式不消失，而是被 reward 近似。

如果教师是 API，你不能计算 \(\pi_T(v|c)\)。这时只能学习一个替代信号：判别器、rubric、verbal score 或 preference reward。

white-box unavailable

不能算 token KL

$$D_{\mathrm{KL}}(\pi_\theta(\cdot|c_t)\|\pi_T(\cdot|c_t))\quad\text{requires teacher logits}$$

黑盒教师只给文本，所以只能比较完整输出或偏好。

reward replacement

用 \(D_\phi\) 学教师品味

$$\max_\theta\ \mathbb E_{y\sim\pi_\theta}D_\phi(x,y)-\beta D_{\mathrm{KL}}(\pi_\theta\|\pi_{\mathrm{ref}})$$

结论：黑盒 OPD 更像 RL；瓶颈从 KL 计算转为 reward 可靠性。

9. 图像生成 OPD：从 token KL 推到 trajectory matching

图像模型的“下一步”不是 token，而是 latent 状态。

这部分是你的重点方向。核心桥梁：把 \(c_t=(x,y_{<t})\) 换成 diffusion/flow 的 \(z_t\)。

diffusion Markov step

从 token 分布换成状态转移分布

语言 OPD 在每个前缀上比较 token 分布：

$$D_{\mathrm{KL}}(\pi_\theta(\cdot|c_t)\|\pi_T(\cdot|c_t))$$

扩散模型在每个 latent state 上比较反向转移分布：

$$D_{\mathrm{KL}}(p_\theta(z_{t-\Delta}|z_t,c)\|p_T(z_{t-\Delta}|z_t,c))$$

学生 rollout 对应 \(z_t\sim p_\theta\) 产生的去噪轨迹。

Gaussian KL

为什么会变成 mean / velocity L2

如果两个反向转移都是同方差高斯：

$$p_\theta=\mathcal N(\mu_\theta,\sigma^2I),\quad p_T=\mathcal N(\mu_T,\sigma^2I)$$

那么 KL 有闭式：

$$D_{\mathrm{KL}}(p_\theta\|p_T)=\frac{1}{2\sigma^2}\|\mu_\theta(z_t,t)-\mu_T(z_t,t)\|_2^2$$

结论：diffusion OPD 的 per-step KL 可以落成均值匹配。

flow / ODE

mean matching 推到 velocity matching

flow 或 ODE 里，一步更新常写成：

$$z_{t-\Delta}=z_t+\Delta v(z_t,t)$$

如果教师和学生只差速度场，那么均值差约等于速度差乘步长：

$$\|\mu_\theta-\mu_T\|_2^2\approx \Delta^2\|v_\theta(z_t,t)-v_T(z_t,t)\|_2^2$$

这就是 pi-Flow / flow OPD 里常见的 trajectory velocity matching。

image OPD objective

最终图像 OPD 公式

把期望放到学生自己的 latent trajectory 上：

$$\mathcal L_{\mathrm{image\ OPD}}= \mathbb E_{z_t\sim\mathrm{student}}\sum_t \left[ D_{\mathrm{KL}}(p_\theta(z_{t-\Delta}|z_t,c)\|p_T(z_{t-\Delta}|z_t,c)) \right]$$

常用近似形式：

$$\mathcal L_{\mathrm{traj}}\propto \mathbb E_{z_t\sim\mathrm{student}}\sum_t \|v_\theta(z_t,t)-v_T(z_t,t)\|_2^2$$

我的判断：图像 OPD 的灵魂不是给最终图像打分，而是在学生中间轨迹上持续纠偏。

10. 最后把所有公式收束成一句话

OPD 的核心公式只有一个骨架，论文都在替换部件。

如果你能看懂下面这个骨架，就能快速读懂大多数 OPD 论文。

统一骨架

$$ \mathcal L_{\mathrm{OPD}}= \mathbb E_{\text{state}\sim\text{student rollout}} \sum_t \mathrm{TeacherSignal}\left(\text{student state at }t\right) $$

语言模型里，state 是 \(c_t=(x,y_{<t})\)，TeacherSignal 是 token KL、top-k KL、advantage 或 reward。图像模型里，state 是 \(z_t\)，TeacherSignal 是 Gaussian KL、mean matching、velocity matching、score matching 或 reward。

结论 1

OPD 修的是分布错位

SFT/KD 多在标准前缀上训练；OPD 在学生真实前缀上训练。

结论 2

KL 方向决定行为

reverse KL 压错误模式，forward KL 保多样性；稳定方法本质是在调这个取舍。

结论 3

图像 OPD 是轨迹纠偏

token-level guidance 迁移到 latent trajectory 后，经常变成 mean / velocity / score matching。
